Timer — My First Web Project
A lightweight, browser‑based countdown timer built with HTML, CSS, and JavaScript. This project began as my very first experiment in front‑end development and static web hosting, created while I was learning Azure and building the foundations of my portfolio. Over time it grew, evolved, and picked up new features as my skills developed.
Originally, this timer was my landing page — the first thing anyone saw when visiting my site, and my first ever Azure Static Web App. After many months of learning, refining, and expanding my portfolio, the timer has now been happily retired to its own dedicated repository and is linked to from my main site as a completed project.

🌱 Project Background
This timer represents the starting point of my journey into:
- Front‑end development
- Static web hosting
- Azure services
- GitHub workflows
- UI/UX experimentation
- Iterative learning over many months
It was built with the guidance of GitHub Copilot, who acted as a patient and surprisingly insightful coding colleague throughout the process. Every improvement I made to this timer reflected something new I had learned — from basic DOM manipulation to styling, animations, and responsive layout.

✨ Features
- Clean, simple countdown interface
- Customisable timer duration
- Responsive layout for desktop and mobile
- Lightweight, dependency‑free implementation
- Evolved design and structure shaped by months of learning

🧱 Tech Stack
- HTML — structure and layout
- CSS — styling, animations, responsive design
- JavaScript — timer logic and UI interactions

🧩 Lessons Learned
Building this timer over many months taught me far more than I expected when I first wrote a few lines of JavaScript to make numbers count down. Some of the most valuable lessons came from the smallest features.
- How browsers cache static assets — Early on, I couldn’t understand why my CSS changes weren’t appearing. This led me to learn about browser caching, cache‑busting query parameters, and how versioning (?v=1.0) forces the browser to fetch fresh files.
- How favicons actually work — Updating the favicon became a whole learning journey. I discovered how browsers handle different icon formats, how to reference them correctly in HTML, and how to avoid the “old icon stuck forever” problem. This was one of the first times I realised how much detail sits behind seemingly simple features.
- How to structure a project cleanly — As the timer grew, I learned to separate HTML, CSS, JavaScript, and assets into a clear folder structure. This made the code easier to maintain and eventually made it possible to move the project into its own repo.
- How to iterate over time — The timer wasn’t built in one go. It evolved as I learned more about Azure, GitHub, and front‑end development. Each improvement reflected something new I had picked up along the way.
- How to work with Copilot as a coding partner — This was the first project where I leaned on Copilot for guidance. It helped me understand patterns, spot mistakes, and learn by example. It wasn’t doing the work for me — it was acting like a colleague who always had a suggestion ready when I got stuck.
- How to deploy a static site — Hosting the timer as my first Azure Static Web App taught me the basics of deployment, file paths, and how the structure of a repo affects what gets served.
- How to transition a project gracefully — When my portfolio grew, I learned how to retire the timer from being the landing page and give it a proper home as a standalone project. This was the first time I treated something I built as part of a larger ecosystem.
These lessons make the timer more than a simple tool — it’s a record of where I started and how far I’ve come.
